4.6	List Main Modules / Components / Processes / Techniques of New System / Proposed System

The proposed system (eSociety – Society Management System) developed during my internship at Grownited Web Development Company consists of several important modules. Each module plays a key role in ensuring smooth functioning, better user experience, and efficient society administration.

User Module:
This module manages user registration, login, profile handling, and role-based access control. It supports multiple roles including Residents, Security Personnel, and Administrators. It ensures that only authenticated users can access system features, improving security and personalization.

Resident Module:
This module handles all resident-related information such as flat/unit details, contact information, family members, and vehicle registrations. It provides a centralized profile for each resident within the society.

Finance Module:
This module manages all financial operations including maintenance fee collection, payment tracking, and financial reporting. It supports online payment integration, automated invoice generation, and provides residents with a clear view of their due and paid amounts.

Visitor Management Module:
This module allows security personnel and residents to log, track, and manage visitor entries and exits. It maintains a digital security log, enabling better monitoring and improving the overall safety of the society premises.

Facility / Amenity Booking Module:
Residents can book shared society facilities such as the clubhouse, swimming pool, or banquet hall through this module. It handles booking requests, availability management, and administrative approval workflows.

Complaint / Issue Tracking Module:
This module enables residents to raise complaints or maintenance requests (plumbing, electrical, etc.). Each complaint is tracked from submission to resolution, ensuring accountability and timely service.

Notice Board / Announcement Module:
Society administrators can post important announcements, circulars, and notices through this module. It ensures all residents are informed about upcoming events, maintenance schedules, and society rules.

AI Chatbot Module:
One of the most advanced components of the system. It integrates an AI-powered chatbot that allows residents to interact with the system in natural language — querying account details, checking facility availability, filing complaints, and managing bookings directly through the chat interface.

Admin Module:
The admin module provides a comprehensive dashboard for society administrators to manage users, approve payments, verify facility bookings, monitor complaints, and oversee all society operations. It ensures smooth, controlled, and transparent management of the platform.

Security & Access Control Module:
This module enforces role-based permissions throughout the system. It ensures that residents, security staff, and admins each have access only to the features and data relevant to their roles, maintaining data privacy and system integrity.

4.7	Selection of Hardware / Software / Algorithms / Methodology / Techniques / Approaches and Justification

The proposed system (eSociety – Society Management System) developed during my internship at Grownited Web Development Company requires a combination of suitable hardware, software, algorithms, and development methodology to ensure efficient performance, scalability, and reliability.

Selection of Hardware

Computer System:
A desktop or laptop is required for designing, developing, and testing the system. A system with a good processor and sufficient memory ensures smooth development and execution of the application.
•	Processor: Intel Core i5 or i7
•	RAM: Minimum 8 GB or more
•	Storage: At least 256 GB SSD
•	Graphics: Integrated or dedicated GPU for smooth performance

Monitor:
A high-resolution monitor (24–27 inches, Full HD or higher) is recommended for better visibility while coding and designing user interfaces. It improves productivity and reduces eye strain during long development sessions.

Server:
A server is required to host the application and manage backend operations. It ensures that users (residents, security staff, and admins) can access the system anytime with stable performance.
•	Processor: Intel Xeon or equivalent
•	RAM: Minimum 8 GB
•	Storage: 500 GB or higher
•	Bandwidth: High-speed internet connection

Network Equipment:
Routers, switches, and stable internet connectivity are required to ensure smooth communication between client, server, and database systems. This is particularly important for real-time notifications and visitor management features.

Backup System:
A backup system is necessary to prevent data loss. Cloud storage or external drives are used to store important project files and database backups regularly, ensuring residents' financial and personal data is always protected.

Development Tools:
Tools like IDEs (VS Code), browsers, and version control systems (Git) are required for efficient development, testing, and collaboration during the project lifecycle.

Selection of Software

Operating System:
The system can be developed and deployed on Windows, Linux, or macOS. These operating systems provide stable environments for software development and server hosting.

Web Server:
Servers like Gunicorn (for Django deployment) combined with Nginx or Apache are used to host and manage the application efficiently in production environments.

Database:
PostgreSQL or SQLite (for development) is used as the relational database to store user data, financial records, visitor logs, facility bookings, complaints, and announcements in a structured and organized manner.

Server-Side Framework:
Django (Python) is used as the primary backend framework. It provides built-in features for authentication, ORM-based database management, admin panel, form handling, and security, making it ideal for a data-intensive society management system.

Front-End Technologies:
HTML5, CSS3, Bootstrap, and JavaScript are used for building the user interface. They enable the creation of responsive, visually appealing, and interactive dashboards for residents and administrators.

IDE (Development Tools):
Visual Studio Code is used as the primary code editor. It supports extensions for Python/Django, debugging, and Git integration, making development efficient and organized.

Version Control System:
Git is used for tracking code changes and collaboration. It helps maintain different versions of the project and ensures a safe development workflow via platforms like GitHub.

AI Integration:
The Google Gemini API is integrated into the system to power the AI chatbot module, enabling natural language processing for user interactions, intent recognition, and intelligent query handling.

Website Analytics (Optional):
Tools such as Django's built-in logging or third-party analytics can be used to monitor system usage, track user activity, and identify areas for future improvement.

Algorithms Used

Role-Based Access Control (RBAC) Algorithm:
This algorithm is used to manage permissions and access rights for different user roles (Resident, Security, Admin). It ensures each user can only access the data and features relevant to their role, maintaining security and privacy.

Search and Filter Algorithm:
Search and filtering techniques are applied to complaints, visitor logs, financial records, and facility bookings. They help administrators and residents quickly locate relevant records based on criteria like date, category, or status.

AI Intent Recognition Algorithm:
The integrated AI chatbot uses natural language processing and intent classification to understand user queries and map them to appropriate system actions such as checking maintenance dues, filing complaints, or booking a facility.

Approval Workflow Algorithm:
A state-machine-based workflow algorithm manages the lifecycle of financial transactions and facility bookings — tracking them through states such as Pending, Approved, and Rejected — ensuring proper administrative oversight.

Data Analysis Techniques:
Data analysis is used to generate financial summaries, monitor complaint resolution rates, and understand usage patterns of society facilities. It helps administrators make informed decisions for society management.

Natural Language Processing (NLP):
NLP techniques, powered by the Gemini API, enable the chatbot to process and understand user inputs in natural language, making the system more accessible and user-friendly for all residents.

Methodology

Agile Methodology:
The development of the eSociety system follows the Agile model, where the project is divided into small, manageable iterations (sprints). Each sprint focuses on developing, testing, and refining specific modules — such as Finance, Visitor Management, or the AI Chatbot.

This methodology allows continuous feedback from stakeholders, faster delivery of working features, and the flexibility to adapt to changing requirements throughout the project. It is highly suitable for a multi-module, real-world web application like eSociety where requirements evolve iteratively.
